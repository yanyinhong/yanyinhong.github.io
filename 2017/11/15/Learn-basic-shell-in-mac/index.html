Mac OS下Shell的学习

11月 15, 2017 发布在 前端

Shell是Linux系统下的一个交互式工具，可以帮助用户启动程序、管理文件系统中的文件以及管理运行在系统上的进程。

用户可以通过输入命令来新建，复制，移动，删除文件等。还可以将多条命令写入文件中，通过执行这个脚本文件来进行操作。对开发人员来说，是一个帮忙提升工作效率的好工具。

Mac的操作系统Mac OS是基于unix衍生来的系统，默认安装Shell。对应Windows下的cmd。下面记录一下Mac下Shell的常见使用方法。

调出Shell

通过Mac的Terminal使用Shell命令。

方法1

通过点击桌面下方Dock中的“Finder =》应用程序 =》终端”，调出终端窗口。

方法2

点击快捷键“command + 空格”，调出Spotlight。输入“终端”或“terminal”，回车，调出终端窗口。

Shell版本

Mac OS X 10.2之前默认的是C Shell，Mac OS X 10.3之后默认的是Bourne Shell。

如果想确认系统的Shell版本，打开Mac的终端，输入：

如果输出结果是csh或者tcsh，那么系统安装的是C shell；如果输出结果是base，sh，zsh，那么系统安装的可能是Bourne Shell的一个变种。

文件系统结构

Mac OS X是经过Unix03认证的Unix系统。它的目录结构遵循Unix系统的文件结构，下面列举一些目录的顶层结构及功能。

目录 用途
/ 虚拟目录的根目录。通常不会在这里存储文件
/bin 二进制目录，存放许多用户级的GNU工具
/boot 启动目录，存放启动文件
/dev 设备目录，Linux在这里创建设备节点
/etc 系统配置文件目录
/home 主目录，Linux在这里创建用户目录
/lib 库目录，存放系统和应用程序的库文件
/media 媒体目录，可移动媒体设备的常用挂载点
/mnt 挂载目录，另一个可移动媒体设备的常用挂载点
/opt 可选目录，常用于存放第三方软件包和数据文件
/proc 进程目录，存放现有硬件及当前进程的相关信息
/root root用户的主目录
/sbin 系统二进制目录，存放许多GNU管理员级工具
/run 运行目录，存放系统运作时的运行时数据
/src 服务目录，存放本地服务的相关文件
/sys 系统目录，存放系统硬件信息的相关文件
/tmp 临时目录，可以在该目录中创建和删除临时工作文件
/usr 用户二进制目录，大量用户级的GNU工具和数据文件都存储在这里
/var 可变目录，用以存放经常变化的文件，比如日志文件

常见的目录名都是基于文件系统层级标准（Filesystem Hierarchy Standard，FHS）的。FHS偶尔会有更新，如果想要查询最新标准，可以查询FHS官网：http://www.pathname.com/fhs。

Shell常用命令

使用man命令可以查询存储在系统上的命令手册页面。

下面记录一些常用的命令

用法 说明
cd www 进入当前目录下的www
pwd 查询当前所在的目录
ls 显示当前目录下的文件和目录
ls -F 在目录后加/，方便区分目录
ls -a 隐藏文件和显示文件一起显示
ls -R 递归显示文件，显示子目录下的文件
ls -l 显示文件的相关信息
touch file 创建名为file的文件。
cp source destination 复制
cp -R dir/ newdir 递归复制dir目录中的内容到新目录newdir
ln -s file sl_file 创建符号链接sl_file，链接指向file
ln file hl_file 创建硬链接hl_file，链接指向file
mv dir newdir 重命名dir为newdir
rm file -i 删除文件file，并且在删除前询问
rm dir -r 删除目录中的文件以及目录本身
rm dir -f 强制删除
file myfile 查看文件的类型
cat myfile 查看整个文件
cat myfile -n 给文件的每个行加上行号
cat myfile -b 给有文本的行加上行号
cat myfile -T 不显示制表符
more myfile 按页显示文本
less myfile 按页显示文本，并提供更高级的特性
tail myfile 查看文件最后10行的内容
tail myfile -n 指定显示文本的行数
tail myfile -f 允许在其他进程使用该文件时查看文件的内容
head myfile 查看文件开头10行的内容
head myfile -n 指定显示文本的行数

如果file文件已经存在，touch file将修改file的修改时间，touch file -a则修改file的访问时间。

ls -l显示的时间是文件的修改时间，加上--time=atime参数就能够显示文件的访问时间了。

cp src dest时，如果目标文件已经存在，系统会默认覆盖。最好加上-i参数，强制shell询问是否覆盖已有文件。

符号链接是实实在在的文件，它指向文件结构中的另一个文件；硬链接会创建独立的虚拟文件，其中包含了原始文件的信息及位置，但是它们从根本上而言是同一个文件。

rm dir -rf命令可以递归并强制删除目录和目录中的文件。

cat将会展示所有的内容，不能停止进程。more和less则可以分页展示。

tail命令会保持活动状态，并不断显示添加到文件中的内容。这是实时监测系统日志的绝妙方式。

head不支持-f。
